指定されたPRの変更点について100個の事実を箇条書きでPRコメントとして投稿してください。

## 引数

PR番号: $ARGUMENTS

## 手順

### Step 1: PRの情報を取得

```bash
gh pr view $ARGUMENTS --json title,body,additions,deletions,changedFiles,files
```

### Step 2: PRの差分を取得

```bash
gh pr diff $ARGUMENTS
```

### Step 3: 100個の事実を生成

取得したPRの差分を分析し、以下の観点から100個の事実を箇条書きで生成してください：

- 追加されたファイル
- 削除されたファイル
- 変更されたファイル
- 追加された関数/メソッド
- 削除された関数/メソッド
- 変更された関数/メソッド
- 追加されたインポート
- 削除されたインポート
- 追加された依存関係
- 削除された依存関係
- 追加された型定義
- 変更されたロジック
- 追加されたコメント
- 変更された設定
- 追加されたテスト
- 変更されたテスト
- リファクタリングの内容
- バグ修正の内容
- 機能追加の内容
- その他の変更点

### Step 4: PRコメントとして投稿

生成した100個の事実を以下のフォーマットでPRコメントとして投稿してください：

```bash
gh pr comment $ARGUMENTS --body "$(cat <<'EOF'
## このPRの変更点についての100個の事実

1. ...
2. ...
3. ...
...
100. ...

---
*このコメントはClaude Codeによって自動生成されました*
EOF
)"
```

## 注意事項

- 必ず100個の事実を生成してください。足りない場合は、より細かい粒度で事実を抽出してください。
- 事実は客観的で具体的な内容にしてください。
- 同じ内容の重複は避けてください。
- 差分が小さい場合でも、コードの構造、命名、スタイルなど細かい点にも言及して100個を目指してください。
